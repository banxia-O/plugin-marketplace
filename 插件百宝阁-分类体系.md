# 插件百宝阁 — 分类体系

> 设计原则：大类固定（轻易不改），子类可动态增删。插件上传时选大类（必选1个）+ 子类（可多选）。前端按大类渲染入口，子类作为筛选标签。空子类不显示，有内容自动出现。

---

## 前端展示逻辑

1. **首页分类入口**：只显示大类（12 个），用图标 + 名称卡片展示
2. **分类页**：点击大类后展示该类下所有插件，左侧/顶部显示子类标签作为筛选器
3. **空子类处理**：没有插件的子类不显示标签，有了自动出现
4. **多选标签**：一个插件可以同时属于多个大类+多个子类（如一个医学文献检索插件 = 科研学术/医学 + 信息检索/搜索引擎）

## 数据库设计建议

```
categories 表：
  id          INTEGER PRIMARY KEY
  name        TEXT        -- 大类名称（如"信息检索"）
  slug        TEXT        -- URL 友好标识（如"search"）
  icon        TEXT        -- 图标标识
  sort_order  INTEGER     -- 排序权重

subcategories 表：
  id          INTEGER PRIMARY KEY
  category_id INTEGER     -- 外键关联大类
  name        TEXT        -- 子类名称（如"搜索引擎"）
  slug        TEXT
  sort_order  INTEGER

plugin_categories 表（多对多）：
  plugin_id      INTEGER
  subcategory_id INTEGER
  PRIMARY KEY (plugin_id, subcategory_id)
```

这样大类和子类都可以后台动态管理，插件与分类是多对多关系，灵活不写死。

## V1 建议：先激活哪些子类

根据目前 MCP 生态中插件的实际分布，V1 上线时以下子类大概率能填上内容：

**肯定有内容的**：搜索引擎、网页抓取、文件系统、SQL数据库、即时通讯、邮件、代码仓库、容器与云服务、记忆与知识、天气

**可能有内容的**：文档处理、云存储、NoSQL数据库、项目管理、笔记写作、终端命令行、地图导航、安全扫描

**V1 大概率空的**：社交媒体（国内）、购物比价、出行旅行、大部分科研子类、视频、UI设计

空子类前端自动隐藏即可，不影响体验。
